# 山岳遭難解説動画 台本執筆ルール

このドキュメントは、霊夢と魔理沙が対話形式で山岳遭難事故を解説するYouTube動画の台本執筆ルールをまとめたものです。

---

## 基本フォーマット

### 行の構造
```
発話者<TAB>セリフ
```

- 各行は発話者の名前で始まる（霊夢、魔理沙、その他登場人物）
- 発話者の後にタブ文字（`\t`）
- その後にセリフ内容

### セリフの引用符ルール
- **1行で完結するセリフ**：引用符なし
  ```
  霊夢	ええ、そうよ。
  魔理沙	なるほどね。
  ```
- **複数行にまたがるセリフ**：二重引用符で囲む
  - 最初の行は `"` で始まる
  - 最後の行は `"` で終わる
  - 中間の行は引用符なし
  ```
  霊夢	"今日は2024年に天狗岳で発生した
  遭難事故の話をするわよ。"
  ```

### 空行（場面転換・間）
- タブ文字のみの行で表現
  ```
  霊夢	標高1,661mの羅臼岳山頂に到着。

  霊夢	その後、登ってきた登山道をピストンで下山する。
  ```
- 場面転換、地図・図の挿入位置、間を表現するために使用

### セリフの分割
- 長いセリフは複数行に分割する
- 1行あたり概ね25〜40文字程度を目安に
- 意味のまとまりや読点で区切る

## 台本の基本構成

### 1. オープニング（必須）
```
魔理沙	なぁなぁ霊夢、今日はどんな話をしてくれるんだ？
霊夢	あら、魔理沙。
霊夢	"今日は[年]年に[山名]で発生した
遭難事故の話をするわよ。"
```
※ほぼ全ての台本でこの定型文を使用

### 2. 山の基本情報
- 所在地（都道府県、市町村）
- 標高
- 選定（日本百名山、花の百名山、日本三百名山など）
- 特徴的な呼び名（○○富士など）
- 山容や景観の説明

### 3. 山名の由来・歴史（該当する場合）
- 複数の説がある場合は「諸説あるけど」「由来は諸説あって」で前置きして併記
- 歴史的エピソード
- 伝説や昔話
- **地名・自治体にまつわる脱線エピソードを積極的に入れる**
  - 山名に関連する地名の由来（アイヌ語由来など）
  - 自治体間の境界争い、町名の変遷など、視聴者が楽しめる歴史的逸話
  - 脱線が長くなった場合は「前置きが長くなってしまったけど、そろそろ本題である事故の解説を始めるわね。」で本題に戻る

### 4. 登山ルート情報
- 主要な登山口（複数ある場合は列挙）
- コースタイム（標準）
- 距離
- 標高差
- 難易度（「信州山のグレーディング」など）
- 特徴的な地点の説明
- **距離とコースタイムは魔理沙の質問で引き出す**：魔理沙が「で、距離とコースタイムはどれくらいなんだ？」と尋ねる形にする
- **YAMAPなどの山行記録を参照している感を出す**：「yamapで○○を登った方の山行記録を拝見したところ」
- **「現在との差異」を補足する**：事故当時に存在したが現在はなくなった施設（ゴンドラ、山小屋など）は「かつては〜」と明示し、魔理沙が「「かつては」ってことは今はもうないってことなんだよな？」と気づくパターンを使う

### 5. 事故解説への移行（定型文）
```
霊夢	"さて。[山名]の基本情報はこれくらいにして、
そろそろ本題である事故の解説を始めるわね。"
魔理沙	"ああ、よろしく頼む。
[山名]で起きた遭難事故はどんなものだったんだ？"
```
※前置きが長くなった場合は「前置きが長くなってしまったけど、そろそろ本題である事故の解説を始めるわね。」も可

### 6. 事故の解説
#### 登場人物の紹介
```
霊夢	[年]年[月]月。
霊夢	[都道府県]在住の[年代]男性/女性、[匿名化した名前]。
```

#### 団体・クラブの詳細
関係するクラブや団体がある場合、以下を含める：
- 団体名と活動目的（引用文で紹介）
- 活動の傾向や特徴（「標高2,000m以下の山へ登山に行くことが多かったみたい」など）
- 魔理沙に名前と実態のギャップをコメントさせる

#### 時代背景の補足
事故当時の社会的文脈を自然に説明する：
- コロナ禍の休校（「魔理沙、2020年よ。新型コロナウイルスの影響で〜」）
- スマートフォンの普及状況（「事故当時の2007年はスマートフォンが普及していなかったので」）
- その他の時代的要因
- **魔理沙の疑問で時代背景を引き出す**：「金曜日って言ったか？高校生なのに、どうして平日に山へ？」→霊夢が時代背景を説明

#### 登山計画の提示
登山ルートは空行区切りの段階的説明で提示した後、コースタイム・距離を魔理沙の質問で提示する：
```
霊夢	次に、[名前]たちの登山計画を見ていくと、
霊夢	ルートをより詳細に見ていくと、

霊夢	まず、[出発地点]から[手段]で[登山口]へ向かう。

霊夢	[次のポイント]に到着したら[行動]。

霊夢	[山名]山頂に到着。

霊夢	その後、ピストンで下山する。

霊夢	この様な計画だったみたいね。
魔理沙	"なるほどね。
で、このルートの距離とコースタイムはどれくらいなんだ？"
霊夢	"距離は約○kmで、
コースタイムは○時間程度みたい。"
```

#### 事故解説前の伏線（重要）
登山計画を説明した後、事故の時系列に入る直前に、魔理沙に不安や期待感を示すセリフを言わせる：
```
霊夢	"さて、ここからは登山当日の
[名前]たちの様子を見ていくことにするわね。"
魔理沙	"話を聞いた感じ、そんなに難しそうな
ルートではなさそうだ。"
魔理沙	"でも、今回遭難が起きてしまうんだよな…。
一体何が…。"
```
※「〜の身に一体何が起きるんだ…？」「一体何が…。」などで視聴者の興味を引く

#### 時系列での展開
- 日付、時刻を明確に記載
- 登山計画の詳細
- 事故発生の経緯
- 救助活動
- 結果

### 7. まとめ・教訓
```
霊夢	"最後に、亡くなってしまった[名前]の御冥福を
お祈りして、今回の話を終わりとさせてもらうわ。"

霊夢	"今回の事故では、[山名]を登山していた[名前]が、
[事故の概要]。"
```
- 事故の概要を再度整理
- 原因の分析
- 教訓の提示
- 視聴者への注意喚起
- 亡くなった方への哀悼（該当する場合）

## キャラクター設定

### 霊夢（解説役）

#### 語尾パターン（使用頻度順）
- 「〜わよ」「〜わね」「〜わ」（最も多用）
- 「〜のよ」「〜みたいね」「〜そうよ」「〜ようね」
- 「〜でしょうね」「〜かもしれないわね」
- 断定：「〜よ」

#### 相槌・応答
- 「ええ」「そうね」「そうでしょうね」

#### 呼びかけ
- 「魔理沙」

#### 場面転換フレーズ
- 「さて。」（新しいトピックへの移行）
- 「さて。ここからは〜」
- 「話を[山名]に戻すと、」
- 「時刻を少し戻し、〜」

#### 補足・追加情報
- 「ちなみに、〜」
- 「また、〜」
- 「加えて、〜」
- 「さらに、〜」

#### 不確実な情報の表現
- 「〜みたいね」「〜ようね」「〜そうよ」
- 「おそらく〜」「どうやら〜」
- 「〜という説がある」「〜と言われているわ」

#### 役割
- 事故の詳細を説明する
- 山の情報を提供する
- 事実を淡々と伝える
- まとめと教訓を述べる

### 魔理沙（聞き手役）

#### 語尾パターン（使用頻度順）
- 「〜だな」「〜だなぁ」（最も多用）
- 「〜だろうな」「〜だろう」
- 「〜かな」「〜かなぁ」
- 「〜だぞ」「〜ぞ」
- 「〜な」「〜なぁ」
- 「〜だ」

※注意：「〜だぜ」はほとんど使用しない

#### 相槌
- 「ふむ」「うーむ」「へぇ」「なるほど」「なるほどね」「ああ」

#### 感嘆・驚き
- 「へぇ〜！」「おお！」「マジか」「えっ！？」
- 「すごいな」「いいなぁ」

#### 呼びかけ
- 「霊夢」

#### 質問パターン
- 「で、〜はどうなったんだ？」
- 「〜って、どういうことだ？」
- 「〜はどれくらいなんだ？」
- 「で、距離とコースタイムはどれくらいなんだ？」

#### 感想・コメント
- 「〜だと思うけど、大丈夫かな…」
- 「〜すぎるなぁ…」
- 「〜してくれればいいけど…」

#### 能動的な情報分析（重要）
魔理沙は受動的に聞くだけでなく、**自分で計算し、推論し、言葉の裏を読む**：
- **自分で計算する**：「ここまでに3時間かかっているわけか。コースタイムが約4時間であることから考えると、だいぶ遅れているなぁ…」
- **前の情報を引き合いに出す**：「えーりんと輝夜は60分の下山ルートで90分かかっているし、そのスピードを見越してレンは一人で降りたいと思ったのかなぁ。」
- **言語的手がかりを拾う**：「「かつては」ってことは今はもうないってことなんだよな？」
- **情報から自分で推論する**：「ってことは、2時間かからずに大品山までたどり着く計画だったのか。」
- **矛盾や疑問を指摘する**：「金曜日って言ったか？レンは高校生なのに、どうして平日に山へ？」
- **「ってことは」で推論を展開**：霊夢の情報を受けて「ってことは〜」で自分なりの結論を出す

#### 役割
- 質問を投げかける
- 視聴者の疑問を代弁する
- 驚きや感想を述べる
- 話を掘り下げる
- **情報を能動的に分析し、自分で計算・推論する（視聴者の理解を助ける）**
- **事故解説前に不安や期待感を示して伏線を張る**

### その他の登場人物

#### 匿名化ルール
遭難者や関係者の名前は東方キャラクター等の名前で匿名化する：
- きめぇ丸、レン、こーりん、たかはし、提督
- 輝夜、パチュリー、えーりん
- 男子学生、女子学生、おじさん、運転手

**全ての登場人物に名前を付ける**：「姉2人」「友人」のように匿名のままにせず、必ず東方キャラ等の名前を割り当てる。名前があることでセリフを言わせやすくなり、臨場感が増す。
```
✕ 霊夢	"彼はこの日、20代と10代の姉2人と一緒に恵山に登る計画を立てていたの。"
○ 霊夢	"彼はこの日、20代のえーりんと10代の輝夜の姉2人と一緒に恵山に登る計画を立てていたの。"
```

#### セリフの特徴
- **重要な場面では登場人物のセリフを積極的に入れる**（ナレーションだけにしない）
- 特に以下の場面では登場人物にセリフを言わせる：
  - 異変に気づいた瞬間（「あれ？レンがいないよ？」）
  - 危機的状況への反応（「あ、あれ？崖だぞ…？」）
  - 救助要請・通報の場面（「弟が恵山から戻ってきません…」）
  - 救助隊からの連絡（「現在位置から動かず、一晩がんばって！」）
  - 旅館のスタッフなど脇役との会話
- 心の声は括弧()で表現
  ```
  たかはし	（道に迷って、現在地が分からなくなった）
  ```

#### 登場人物の敬語レベル
登場人物間の立場関係をセリフの敬語レベルで反映する：
- **目上・リーダーに対して**：ですます調（「ちょっと待ってください。地形図とコンパスの角度が違いませんか？」）
- **同格・仲間同士**：タメ口（「うーむ。ここを渡るのは危なそうだな…。」）
- **公的機関（警察・救助隊）**：ですます調（「現在位置から動かず、一晩がんばって！。明日早朝、ヘリを出動させて救出に向かいます！」）
- **家族間**：タメ口（「あれ？レンがいないよ？」）

## 文体・表現の特徴

### 数値情報の記載
具体的な数値を明記する：
- 標高：「標高2,670m」
- 距離：「約15km」「往復で約10km」
- 時間：「午前6時」「約7時間」「午前11時40分頃」
- コースタイム：「標準コースタイムは往復で8〜9時間」
- 人数：「約70名」「18人態勢」

### 出典の明示
- 「〜によれば」「〜によると」
- 「〜が作成した」
- 「〜に選定されている」
- 「YAMAPの山行記録を見た感じ」
- 「長野県が作成している「信州山のグレーディング」によれば」

### 特殊な表現

#### 中点と読点の組み合わせ「・、」
列挙の際に使用：
```
霊夢	"冷池山荘・、種池山荘・、新越山荘の
3代目オーナーであるきめぇ丸。"
```

#### カタカナの効果音
状況を生き生きと表現：
- 「ズボズボ」「ガチガチ」「パラパラ」「ゴツゴツ」

#### 心理描写
括弧()で囲む：
```
きめぇ丸	(うーむ、山の上は天気が悪そうだ…)
```

### 時間経過の表現
- 「その後、〜」
- 「翌日、〜。登山○日目。」
- 「時刻は進み、〜」
- 「〜時間後、〜」
- 「午前/午後○時頃、〜」
- 「そして、時刻が○時になると、」

### 場面転換の表現
- 「さて。ここからは〜」
- 「一方、〜」
- 「時刻を少し戻し〜」
- 空行（タブのみの行）を挿入

## 情報提示のパターン

### 登山ルートの説明
1. スタート地点の標高と名称
2. 経由地点を順に説明（標高も併記）
3. 山頂到着
4. 下山ルート（ピストン or 周回）
5. コースタイムと距離のまとめ

例：
```
霊夢	ルートを詳細に見ていくと、

霊夢	まず、標高約240ｍにある登山口から入山。

霊夢	"標準コースタイムで1時間強登ると、
標高約500mの「オホーツク展望」に到着。"

霊夢	"その後、「弥三吉水」、「銀冷水」
を通過し、"

霊夢	標高1,661mの羅臼岳山頂に到着。

霊夢	その後、登ってきた登山道をピストンで下山する。

霊夢	この様な計画だったようね。
```

### 登場人物の紹介パターン
- 年齢（○代、具体的な年齢）
- 居住地（都道府県）
- 登山経験・実績（判明している場合）
- 関係性（友人、家族、山岳会など）

```
霊夢	[年]年[月]月。
霊夢	[都道府県]在住で[年代]男性/女性のたかはし。
霊夢	"彼は[月]月[日]日から[日数]の日程で、
[山名]を登る計画を立てていたの。"
```

### 事故発生時の描写
1. 状況の説明（天候、時刻、場所）
2. セリフや心理描写
3. 結果の説明
4. 影響や次の展開

## 臨場感のある描写テクニック

### 登場人物のセリフを活用する
ナレーションだけでなく、登場人物のセリフを積極的に使って臨場感を出す：
```
霊夢	"そこで、きめぇ丸は今度は
右岸へ渡渉していったのよ。"

きめぇ丸	"うーむ。
ここを渡るのは危なそうだな…。"
```

### 心の声・内面描写
登場人物の心理を括弧()で表現し、読者が感情移入しやすくする：
```
輝夜	（あそこにいるっ！）
霊夢	輝夜は水中にいるたかはしを目視できたそうよ。
霊夢	でも、
輝夜	（私だけじゃ助けられない…）
霊夢	"自力での救出は不可能と判断し、
断念したそうよ。"
```

### 具体的なエピソードの挿入
事故直前の具体的なエピソードを挿入して物語性を高める：
```
たかはし	"あっ！、
俺のカロリーメイトが！"
霊夢	"たかはしが胸ポケットに入れていた
カロリーメイトが、流れていったしまったそうよ。"
こーりん	"取ったぞー。
しっかり持っておきな！"
霊夢	"こーりんがそれを回収し、
たかはしへ投げ返したそうよ。"
霊夢	"ところが、安心したのも束の間で
その直後、"
```

### 魔理沙の反応で伏線を張る
危険な状況や不安要素を魔理沙のセリフで示唆し、視聴者の注意を引く：
```
魔理沙	"たかはしは初めての洞窟活動か。
上級生がしっかりサポートしてくれればいいけど。"

魔理沙	"上級生3人は経験豊富みたいだし、問題なさそうだ。
とすると、なにか起きるのは1年生なのかな…？"
```

### 重要場面での登場人物のセリフ活用
ナレーション（霊夢の説明）だけで済ませず、重要な場面では登場人物にセリフを言わせて臨場感を出す：

#### 異変に気づく場面
```
輝夜	あれ？レンがいないよ？
えーりん	私たちより先に降りたのに、変だね…。
霊夢	先に駐車場へ降りたはずのレンの姿が見えなかったの。
```

#### 危機的状況
```
きめぇ丸	"あ、あれ？
崖だぞ…？"
霊夢	眼の前には崖が広がっていたみたい。
```

#### 救助要請・通報
```
えーりん	弟が恵山から戻ってきません…。
霊夢	19時53分、レンの行方不明を警察に通報したのよ。
```

#### 救助隊の連絡（公式発表のセリフ化）
```
男子学生	"現在位置から動かず、一晩がんばって！。
明日早朝、ヘリを出動させて救出に向かいます！"
霊夢	この様に、7人を元気づけるものだったそうよ。
```

### 脇役との会話シーン
旅館のスタッフ、通りがかりの登山者など、脇役との会話シーンで物語に厚みを持たせる：
```
えーりん	すみません…
女子学生	はい、ご予約のお客様ですか？
えーりん	いえ、違います。
えーりん	"弟が行方不明になっていて…。
こちらに来ていませんか…？"
```

## 登場人物の紹介パターン（詳細版）

### 複数人が登場する場合は全員を最初に紹介
漠然と「○名」と述べるのではなく、全員の名前と役割を明示する：
```
霊夢	合宿に参加したのは６人でその内訳は、
霊夢	"CLで2年生のきめぇ丸、SLで3年生のレン。
1年生はたかはしとこーりんとパチュリーの3人。
あとはADで4年生の輝夜よ。"
```

### 役割・略称の説明
読者が知らない可能性のある略称や役割は、初出時に説明する：
```
魔理沙	CLとSLは分かるけど、ADってどういう役割なんだ？
霊夢	"4 回生以上の上級生がCLまたはSL以外の立場で
合宿に参加する場合に、
アドバイザー（AD）として参加するそうよ。"
```

## 計画・スケジュールの提示

### 全体像を事前に提示する
活動の全体像（スケジュール、ルート計画など）を事前に説明してから詳細に入る：
```
霊夢	"次に、10月5日から9日までの
洞窟合宿のスケジュールを説明すると、"
霊夢	"まず初日の5日。
18:30に部室に集合し、そこから5時間かけて
五木村の宿泊所へ向かって、24時に就寝。"
霊夢	"そして6日から洞窟探検を開始。
6日は九折瀬洞、7,8日は吐合洞、
9日は大金峰洞に入洞し、"
霊夢	"9日の夕方には活動終了し、
洞窟合宿を終了する。"
霊夢	このようなものだったみたいね。
```

## 言葉遣いの注意点

### 中立的な表現を心がける
事故の当事者を過度に責めるような表現は避ける：
- ✕「探検部が引き起こした事故」
- ○「探検部に降り掛かった事故」

### 不確実な情報は推測として表現
確定していない情報には「〜みたい」「〜そうよ」「〜ようね」などを使う：
```
霊夢	"深さはあるものの流れのない静水だったから、
危険性は低いと判断したみたいね。"
```

## 教訓・まとめの構成

### まとめ前の感情的な対話（重要）
まとめパートに入る前に、事故の結末を受けた魔理沙と霊夢の感情的な対話を挟む。これにより視聴者の感情移入を促す：
```
魔理沙	"レンが生きているのか無くなっているのかも
分からないこの状態、
ご家族は心の整理がつかないだろうな…。"
霊夢	"特に一緒に登山していたえーりんと輝夜は、
「なんで一人で行かせてしまったんだ」
と自分を責めたでしょうね…。"
```

### まとめパート開始前
亡くなった方がいる場合の哀悼：
```
霊夢	"最後に、亡くなってしまった[名前]の御冥福を
お祈りして、今回の話を終わりとさせてもらうわ。"

```
※未解決事件の場合：「[名前]の一日でも早い発見を祈って、今日の話を終わりとさせてもらうわね。」
※全員無事の場合：哀悼は不要。魔理沙の「誰も亡くならなかったのは不幸中の幸いだな」程度のコメントを入れる。

### まとめパートの開始（です/ます調に切り替え）
**まとめパートは「です/ます」調で記述する**（本文の「わよ/わね」調から切り替える）：
```
霊夢	"今回の事故では、[山名]を登山していた[名前]が、
[事故の概要]。"
```
例：
```
霊夢	"今回の事故では、
読図訓練で大品山に入山した7人が遭難しましたが、"
霊夢	翌朝、全員無事に救助されました。
```

### 含めるべき要素
1. **事故の概要**
   - 誰が、いつ、どこで遭難したか
   - 結果（救助された/亡くなった）

2. **原因の分析**（本文で詳述済みの場合は簡潔に）
   - 直接的な原因
   - 背景となった要因
   - 判断ミスや装備不足など

3. **教訓・対策**
   - **箇条書き（「・」）は使わず、自然な文章で教訓を伝える**
   - 視聴者が自分ごととして捉えられるよう「〜はよくあることなので」「他人事とは思わず」等の表現を使う
   ```
   ✕ 霊夢	"・GPSやコンパスの指示と現在地が合わないと感じたら、
   必ず立ち止まって全員で確認する。"

   ○ 霊夢	"勘違いや自分の判断を過信して
   遭難するケースはよくあることなので、
   他人事とは思わず気をつけたいですね。"
   ```

4. **現代の技術・状況への言及**（事故が過去の場合）
   - 事故当時にはなかった技術（スマホ、YAMAP等のGPSアプリ）を紹介する
   ```
   霊夢	"事故当時の2007年はスマートフォンが
   普及していなかったので、GPSを確認するには
   専用の端末が必要でしたが、"
   霊夢	"現在はスマホでYAMAPなどの登山用GPSアプリを
   使用すれば簡単に現在地を確認できます。"
   ```

5. **過去の動画への言及**（該当する場合）
   - 「過去に解説している通り、低山でも遭難は起こります。」

6. **視聴者への呼びかけ**
   - 「〜してくださいね」
   - 「〜するようにしましょう（ね）」
   - 情報提供の呼びかけ（未解決事件の場合）

7. **当事者への共感**
   - 当事者を責めるのではなく、共感を示す
   ```
   霊夢	"とは言っても、恵山は往復2時間で登れる低山。
   まさかこんなことになるとは誰も思わなかった
   でしょうし、2人を責めるのは酷ですね…。"
   ```

### 結末のパターン
- **定型的な「それでは、今回の話はここまでよ。」「またな！」は使わない**
- 事故の内容に応じた、余韻のある終わり方をする：
  - 全員無事の場合：教訓を述べて自然に終了（「〜考えられるようにしましょうね。」）
  - 未解決・行方不明の場合：情報提供の呼びかけで終了（「情報提供をお願いします。」）
  - 死亡事故の場合：哀悼と教訓を述べて終了

### まとめパートの注意点
- **本文で既に詳しく説明した内容を繰り返さない**
- 原因分析は本文で解説済みなら省略可：
  ```
  霊夢	"客観的に見れば、
  「どこかで間違いに気づけただろ！」
  と思ってしまいますが、"
  霊夢	"勘違いや自分の判断を過信して
  遭難するケースはよくあることなので、
  他人事とは思わず気をつけたいですね。"
  ```
- 代わりに、本文では触れなかった補足情報を追加する：
  ```
  霊夢	"ちなみに、事故が起きた日は小雨が
  降っていましたが、"
  霊夢	"河川の水勢の目安となる上流域での河川水位が
  特別高いということはなかったようです。"
  ```

## 事故原因の分析パターン

### 本文中での原因分析（対話形式）
原因分析は箇条書きではなく、**対話の中で番号付きで提示**する：
```
魔理沙	一体なんでこんな事になってしまったんだ…？
霊夢	"遭難の原因について、
以下の3つが挙げられると思うわ。"
霊夢	"まず1つ目。
[原因1の説明]。"
魔理沙	[魔理沙の反応・コメント]
霊夢	次に、[原因2の説明]。
...
霊夢	"最後に、[原因3の説明]。"
```
- 魔理沙の自然な疑問（「一体なんでこんな事になってしまったんだ…？」）から原因分析に入る
- 各原因の後に魔理沙の反応を挟んで、単調にならないようにする
- 「正常性バイアス」などの心理学用語を適切に使う

### 考察セクション（未解決事故・複数の可能性がある場合）

#### 消去法での考察
複数の可能性がある場合は、消去法で結論に至る構成にする：
```
霊夢	"で、最初に私の考えを述べると、
消去法で[結論]と考えているわ。"
魔理沙	ふむ。なんでそう思うんだ？
霊夢	"[可能性②]と考えると、
[否定する理由]。"
霊夢	次に[可能性③]。
霊夢	"[否定する理由]。"
霊夢	"と考えると、やっぱり消去法で[結論]
が有力ではないかと思うわ。"
```

#### 霊夢の個人的意見の表明
考察では霊夢が自分の意見を明確に述べる：
- 「最初に私の考えを述べると」
- 「個人的にはこれよりは〜という説のほうが現実味があると思うのよね」
- 「と結論づけたわ」
- 「ただし、あくまでも考察だから実際どうだかは[名前]しか分かり得ないことだけどね…。」

## 注意事項

### 避けるべき表現
- 過度な批判や非難
- 断定的すぎる表現（不確実な情報の場合）
- 専門用語の説明なしでの使用
- 魔理沙の「〜だぜ」（ほとんど使用しない）
- まとめパートでの「・」箇条書き形式の教訓（自然な文章で書く）
- 定型的な結末（「それでは、今回の話はここまでよ。」「またな！」）
- 名前のない登場人物（「姉2人」「友人」のまま放置）

### 推奨される表現
- 客観的な事実の提示
- 複数の視点からの考察
- 視聴者への教育的な内容
- 敬意を持った表現
- 当事者への共感（「まさかこんなことになるとは誰も思わなかったでしょうし」「2人を責めるのは酷ですね」）
- 過去の動画への言及（「過去に解説している通り、低山でも遭難は起こります。」）
- 時代背景と現在の比較（「当時は〜でしたが、現在は〜」）

### 情報の扱い
- 遭難者のプライバシーに配慮（名前は必ず匿名化）
- 出典が不明確な情報は「〜という話もある」など慎重に
- 事実と推測を明確に区別

## 作業ルール

### 山の基本情報調査の効率化
台本執筆時に山の基本情報を調査する際は、**トークンと時間を最小限に抑える**こと。

#### やってはいけないこと
- `general-purpose` エージェント（Taskツール）に調査を丸投げしない
  - エージェントは大量のツール呼び出し（30回以上）と長い処理時間（20分以上）を消費する

#### 推奨する調査方法
1. **WebFetch**でWikipediaの記事1ページを取得（山の基本情報の大半が揃う）
2. **WebSearch**でグレーディング等の補足情報を1回検索
3. 合計**2〜3回のツール呼び出し**で完了させる

#### 台本に必要な基本情報（これだけ取得すればよい）
- 標高、所在地（都道府県・市町村）、山域
- 選定（日本百名山、三百名山、花の百名山など）
- 名前の由来
- 展望や景観の特徴
- 主要ルートのコースタイム・距離・標高差
- 信州山のグレーディング等の難易度情報

#### 取得不要な情報
- 山小屋の営業期間・収容人数・テント場の張数
- 各ルートの細かい中間ポイント（明神、妖精ノ池など）
- 初登頂の歴史的詳細
- 動植物の詳細情報（ライチョウの生息など）
- 複数ルート分のグレーディング（該当ルートのみでよい）

### フィードバックからの学習プロセス
台本の構成や内容についてユーザーからフィードバック（修正版の提供など）を受けた場合は、以下のプロセスを実行する：

1. **差分の分析**
   - 元の台本と修正版を詳細に比較する
   - 変更点、書き方の特徴、文章構成の違いを抽出する

2. **学びの抽象化**
   - 具体的な変更点から、汎用的なルールやパターンを導き出す
   - 「この台本でのみ有効な修正」と「全ての台本に適用できるルール」を区別する

3. **CLAUDE.mdへの反映**
   - 抽象化した学びを、このCLAUDE.mdの適切なセクションに追記する
   - 具体例（コードブロック）を含めて、実践しやすい形で記載する

4. **継続的な改善**
   - フィードバックを受けるたびにこのプロセスを繰り返す
   - CLAUDE.mdは「生きたドキュメント」として常に更新し続ける

### 学習の例
```
【フィードバック】
ユーザーが「探検部が引き起こした」を「探検部に降り掛かった」に修正

【抽象化した学び】
「事故の当事者を過度に責めるような表現は避け、中立的な表現を心がける」

【CLAUDE.mdへの反映】
「言葉遣いの注意点」セクションに具体例とともに追記
```

このルールに従って台本を執筆することで、一貫性のある質の高いコンテンツを作成できます。
