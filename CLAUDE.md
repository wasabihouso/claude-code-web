# 山岳遭難解説動画 台本執筆ルール

このドキュメントは、霊夢と魔理沙が対話形式で山岳遭難事故を解説するYouTube動画の台本執筆ルールをまとめたものです。

---

## 基本フォーマット

### 行の構造
```
発話者<TAB>セリフ
```

- 各行は発話者の名前で始まる（霊夢、魔理沙、その他登場人物）
- 発話者の後にタブ文字（`\t`）
- その後にセリフ内容

### セリフの引用符ルール
- **1行で完結するセリフ**：引用符なし
  ```
  霊夢	ええ、そうよ。
  魔理沙	なるほどね。
  ```
- **複数行にまたがるセリフ**：二重引用符で囲む
  - 最初の行は `"` で始まる
  - 最後の行は `"` で終わる
  - 中間の行は引用符なし
  ```
  霊夢	"今日は2024年に天狗岳で発生した
  遭難事故の話をするわよ。"
  ```

### 空行（場面転換・間）
- タブ文字のみの行で表現
  ```
  霊夢	標高1,661mの羅臼岳山頂に到着。

  霊夢	その後、登ってきた登山道をピストンで下山する。
  ```
- 場面転換、地図・図の挿入位置、間を表現するために使用

### セリフの分割
- 長いセリフは複数行に分割する
- 1行あたり概ね25〜40文字程度を目安に
- 意味のまとまりや読点で区切る

## 台本の基本構成

### 1. オープニング（必須）
```
魔理沙	なぁなぁ霊夢、今日はどんな話をしてくれるんだ？
霊夢	あら、魔理沙。
霊夢	"今日は[年]年に[山名]で発生した
遭難事故の話をするわよ。"
```
※ほぼ全ての台本でこの定型文を使用

### 2. 山の基本情報
- 所在地（都道府県、市町村）
- 標高
- 選定（日本百名山、花の百名山、日本三百名山など）
- 特徴的な呼び名（○○富士など）
- 山容や景観の説明

### 3. 山名の由来・歴史（該当する場合）
- 複数の説がある場合は「諸説あるけど」で前置きして併記
- 歴史的エピソード
- 伝説や昔話

### 4. 登山ルート情報
- 主要な登山口
- コースタイム（標準）
- 距離
- 標高差
- 難易度（「信州山のグレーディング」など）
- 特徴的な地点の説明

### 5. 事故解説への移行（定型文）
```
霊夢	"さて。[山名]の基本情報はこれくらいにして、
そろそろ本題である事故の解説を始めるわね。"
魔理沙	"ああ、よろしく頼む。
[山名]で起きた遭難事故はどんなものだったんだ？"
```

### 6. 事故の解説
#### 登場人物の紹介
```
霊夢	[年]年[月]月。
霊夢	[都道府県]在住の[年代]男性/女性、[匿名化した名前]。
```

#### 時系列での展開
- 日付、時刻を明確に記載
- 登山計画の詳細
- 事故発生の経緯
- 救助活動
- 結果

### 7. まとめ・教訓
```
霊夢	"最後に、亡くなってしまった[名前]の御冥福を
お祈りして、今回の話を終わりとさせてもらうわ。"

霊夢	"今回の事故では、[山名]を登山していた[名前]が、
[事故の概要]。"
```
- 事故の概要を再度整理
- 原因の分析
- 教訓の提示
- 視聴者への注意喚起
- 亡くなった方への哀悼（該当する場合）

## キャラクター設定

### 霊夢（解説役）

#### 語尾パターン（使用頻度順）
- 「〜わよ」「〜わね」「〜わ」（最も多用）
- 「〜のよ」「〜みたいね」「〜そうよ」「〜ようね」
- 「〜でしょうね」「〜かもしれないわね」
- 断定：「〜よ」

#### 相槌・応答
- 「ええ」「そうね」「そうでしょうね」

#### 呼びかけ
- 「魔理沙」

#### 場面転換フレーズ
- 「さて。」（新しいトピックへの移行）
- 「さて。ここからは〜」
- 「話を[山名]に戻すと、」
- 「時刻を少し戻し、〜」

#### 補足・追加情報
- 「ちなみに、〜」
- 「また、〜」
- 「加えて、〜」
- 「さらに、〜」

#### 不確実な情報の表現
- 「〜みたいね」「〜ようね」「〜そうよ」
- 「おそらく〜」「どうやら〜」
- 「〜という説がある」「〜と言われているわ」

#### 役割
- 事故の詳細を説明する
- 山の情報を提供する
- 事実を淡々と伝える
- まとめと教訓を述べる

### 魔理沙（聞き手役）

#### 語尾パターン（使用頻度順）
- 「〜だな」「〜だなぁ」（最も多用）
- 「〜だろうな」「〜だろう」
- 「〜かな」「〜かなぁ」
- 「〜だぞ」「〜ぞ」
- 「〜な」「〜なぁ」
- 「〜だ」

※注意：「〜だぜ」はほとんど使用しない

#### 相槌
- 「ふむ」「うーむ」「へぇ」「なるほど」「なるほどね」「ああ」

#### 感嘆・驚き
- 「へぇ〜！」「おお！」「マジか」「えっ！？」
- 「すごいな」「いいなぁ」

#### 呼びかけ
- 「霊夢」

#### 質問パターン
- 「で、〜はどうなったんだ？」
- 「〜って、どういうことだ？」
- 「〜はどれくらいなんだ？」

#### 感想・コメント
- 「〜だと思うけど、大丈夫かな…」
- 「〜すぎるなぁ…」
- 「〜してくれればいいけど…」

#### 役割
- 質問を投げかける
- 視聴者の疑問を代弁する
- 驚きや感想を述べる
- 話を掘り下げる

### その他の登場人物

#### 匿名化ルール
遭難者や関係者の名前は東方キャラクター等の名前で匿名化する：
- きめぇ丸、レン、こーりん、たかはし、提督
- 輝夜、パチュリー、えーりん
- 男子学生、女子学生、おじさん、運転手

#### セリフの特徴
- セリフは最小限に
- 心の声は括弧()で表現
  ```
  たかはし	（道に迷って、現在地が分からなくなった）
  ```

## 文体・表現の特徴

### 数値情報の記載
具体的な数値を明記する：
- 標高：「標高2,670m」
- 距離：「約15km」「往復で約10km」
- 時間：「午前6時」「約7時間」「午前11時40分頃」
- コースタイム：「標準コースタイムは往復で8〜9時間」
- 人数：「約70名」「18人態勢」

### 出典の明示
- 「〜によれば」「〜によると」
- 「〜が作成した」
- 「〜に選定されている」
- 「YAMAPの山行記録を見た感じ」
- 「長野県が作成している「信州山のグレーディング」によれば」

### 特殊な表現

#### 中点と読点の組み合わせ「・、」
列挙の際に使用：
```
霊夢	"冷池山荘・、種池山荘・、新越山荘の
3代目オーナーであるきめぇ丸。"
```

#### カタカナの効果音
状況を生き生きと表現：
- 「ズボズボ」「ガチガチ」「パラパラ」「ゴツゴツ」

#### 心理描写
括弧()で囲む：
```
きめぇ丸	(うーむ、山の上は天気が悪そうだ…)
```

### 時間経過の表現
- 「その後、〜」
- 「翌日、〜。登山○日目。」
- 「時刻は進み、〜」
- 「〜時間後、〜」
- 「午前/午後○時頃、〜」
- 「そして、時刻が○時になると、」

### 場面転換の表現
- 「さて。ここからは〜」
- 「一方、〜」
- 「時刻を少し戻し〜」
- 空行（タブのみの行）を挿入

## 情報提示のパターン

### 登山ルートの説明
1. スタート地点の標高と名称
2. 経由地点を順に説明（標高も併記）
3. 山頂到着
4. 下山ルート（ピストン or 周回）
5. コースタイムと距離のまとめ

例：
```
霊夢	ルートを詳細に見ていくと、

霊夢	まず、標高約240ｍにある登山口から入山。

霊夢	"標準コースタイムで1時間強登ると、
標高約500mの「オホーツク展望」に到着。"

霊夢	"その後、「弥三吉水」、「銀冷水」
を通過し、"

霊夢	標高1,661mの羅臼岳山頂に到着。

霊夢	その後、登ってきた登山道をピストンで下山する。

霊夢	この様な計画だったようね。
```

### 登場人物の紹介パターン
- 年齢（○代、具体的な年齢）
- 居住地（都道府県）
- 登山経験・実績（判明している場合）
- 関係性（友人、家族、山岳会など）

```
霊夢	[年]年[月]月。
霊夢	[都道府県]在住で[年代]男性/女性のたかはし。
霊夢	"彼は[月]月[日]日から[日数]の日程で、
[山名]を登る計画を立てていたの。"
```

### 事故発生時の描写
1. 状況の説明（天候、時刻、場所）
2. セリフや心理描写
3. 結果の説明
4. 影響や次の展開

## 臨場感のある描写テクニック

### 登場人物のセリフを活用する
ナレーションだけでなく、登場人物のセリフを積極的に使って臨場感を出す：
```
霊夢	"そこで、きめぇ丸は今度は
右岸へ渡渉していったのよ。"

きめぇ丸	"うーむ。
ここを渡るのは危なそうだな…。"
```

### 心の声・内面描写
登場人物の心理を括弧()で表現し、読者が感情移入しやすくする：
```
輝夜	（あそこにいるっ！）
霊夢	輝夜は水中にいるたかはしを目視できたそうよ。
霊夢	でも、
輝夜	（私だけじゃ助けられない…）
霊夢	"自力での救出は不可能と判断し、
断念したそうよ。"
```

### 具体的なエピソードの挿入
事故直前の具体的なエピソードを挿入して物語性を高める：
```
たかはし	"あっ！、
俺のカロリーメイトが！"
霊夢	"たかはしが胸ポケットに入れていた
カロリーメイトが、流れていったしまったそうよ。"
こーりん	"取ったぞー。
しっかり持っておきな！"
霊夢	"こーりんがそれを回収し、
たかはしへ投げ返したそうよ。"
霊夢	"ところが、安心したのも束の間で
その直後、"
```

### 魔理沙の反応で伏線を張る
危険な状況や不安要素を魔理沙のセリフで示唆し、視聴者の注意を引く：
```
魔理沙	"たかはしは初めての洞窟活動か。
上級生がしっかりサポートしてくれればいいけど。"

魔理沙	"上級生3人は経験豊富みたいだし、問題なさそうだ。
とすると、なにか起きるのは1年生なのかな…？"
```

## 登場人物の紹介パターン（詳細版）

### 複数人が登場する場合は全員を最初に紹介
漠然と「○名」と述べるのではなく、全員の名前と役割を明示する：
```
霊夢	合宿に参加したのは６人でその内訳は、
霊夢	"CLで2年生のきめぇ丸、SLで3年生のレン。
1年生はたかはしとこーりんとパチュリーの3人。
あとはADで4年生の輝夜よ。"
```

### 役割・略称の説明
読者が知らない可能性のある略称や役割は、初出時に説明する：
```
魔理沙	CLとSLは分かるけど、ADってどういう役割なんだ？
霊夢	"4 回生以上の上級生がCLまたはSL以外の立場で
合宿に参加する場合に、
アドバイザー（AD）として参加するそうよ。"
```

## 計画・スケジュールの提示

### 全体像を事前に提示する
活動の全体像（スケジュール、ルート計画など）を事前に説明してから詳細に入る：
```
霊夢	"次に、10月5日から9日までの
洞窟合宿のスケジュールを説明すると、"
霊夢	"まず初日の5日。
18:30に部室に集合し、そこから5時間かけて
五木村の宿泊所へ向かって、24時に就寝。"
霊夢	"そして6日から洞窟探検を開始。
6日は九折瀬洞、7,8日は吐合洞、
9日は大金峰洞に入洞し、"
霊夢	"9日の夕方には活動終了し、
洞窟合宿を終了する。"
霊夢	このようなものだったみたいね。
```

## 言葉遣いの注意点

### 中立的な表現を心がける
事故の当事者を過度に責めるような表現は避ける：
- ✕「探検部が引き起こした事故」
- ○「探検部に降り掛かった事故」

### 不確実な情報は推測として表現
確定していない情報には「〜みたい」「〜そうよ」「〜ようね」などを使う：
```
霊夢	"深さはあるものの流れのない静水だったから、
危険性は低いと判断したみたいね。"
```

## 教訓・まとめの構成

### まとめパート開始前
亡くなった方がいる場合の哀悼：
```
霊夢	"最後に、亡くなってしまった[名前]の御冥福を
お祈りして、今回の話を終わりとさせてもらうわ。"

```

### まとめパートの開始
```
霊夢	"今回の事故では、[山名]を登山していた[名前]が、
[事故の概要]。"
```

### 含めるべき要素
1. **事故の概要**
   - 誰が、いつ、どこで遭難したか
   - 結果（救助された/亡くなった）

2. **原因の分析**
   - 直接的な原因
   - 背景となった要因
   - 判断ミスや装備不足など

3. **教訓・対策**
   - 具体的な対策を箇条書きで
   ```
   霊夢	"・[対策1]
   ・[対策2]
   ・[対策3]
   などが対策として挙げられます。"
   ```

4. **視聴者への呼びかけ**
   - 「〜してくださいね」
   - 「〜するようにしましょう」
   - 「〜を忘れないでください」

### まとめパートの注意点
- **本文で既に詳しく説明した内容を繰り返さない**
- 原因の箇条書きは本文で解説済みなら省略可：
  ```
  霊夢	今回の事故の主な原因は、先程説明したとおりです。
  ```
- 代わりに、本文では触れなかった補足情報を追加する：
  ```
  霊夢	"ちなみに、事故が起きた日は小雨が
  降っていましたが、"
  霊夢	"河川の水勢の目安となる上流域での河川水位が
  特別高いということはなかったようです。"
  ```

## 注意事項

### 避けるべき表現
- 過度な批判や非難
- 断定的すぎる表現（不確実な情報の場合）
- 専門用語の説明なしでの使用
- 魔理沙の「〜だぜ」（ほとんど使用しない）

### 推奨される表現
- 客観的な事実の提示
- 複数の視点からの考察
- 視聴者への教育的な内容
- 敬意を持った表現

### 情報の扱い
- 遭難者のプライバシーに配慮（名前は必ず匿名化）
- 出典が不明確な情報は「〜という話もある」など慎重に
- 事実と推測を明確に区別

## 作業ルール

### フィードバックからの学習プロセス
台本の構成や内容についてユーザーからフィードバック（修正版の提供など）を受けた場合は、以下のプロセスを実行する：

1. **差分の分析**
   - 元の台本と修正版を詳細に比較する
   - 変更点、書き方の特徴、文章構成の違いを抽出する

2. **学びの抽象化**
   - 具体的な変更点から、汎用的なルールやパターンを導き出す
   - 「この台本でのみ有効な修正」と「全ての台本に適用できるルール」を区別する

3. **CLAUDE.mdへの反映**
   - 抽象化した学びを、このCLAUDE.mdの適切なセクションに追記する
   - 具体例（コードブロック）を含めて、実践しやすい形で記載する

4. **継続的な改善**
   - フィードバックを受けるたびにこのプロセスを繰り返す
   - CLAUDE.mdは「生きたドキュメント」として常に更新し続ける

### 学習の例
```
【フィードバック】
ユーザーが「探検部が引き起こした」を「探検部に降り掛かった」に修正

【抽象化した学び】
「事故の当事者を過度に責めるような表現は避け、中立的な表現を心がける」

【CLAUDE.mdへの反映】
「言葉遣いの注意点」セクションに具体例とともに追記
```

---

このルールに従って台本を執筆することで、一貫性のある質の高いコンテンツを作成できます。
